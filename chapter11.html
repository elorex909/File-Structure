Hashing

Advanced Searching Techniques, Hash Functions, Collision Resolution, and Performance Analysis

Chapter Overview

This chapter explores hashing as an advanced searching technique for implementing indexes. It covers hash functions, collision reduction strategies, various collision resolution techniques (progressive overflow, buckets, double hashing), and practical implementation issues including deletions and performance analysis.

What is Hashing?

Introduction

Hashing is a powerful searching technique primarily used for implementing indexes. The main motivation is to improve search time efficiency compared to other methods:

Method Time Complexity Description
Simple Indexes (Binary Search) O(log₂N) Traditional indexing with logarithmic search time
B-Trees and B+ Trees O(log₄N) Balanced tree structures for efficient searching
Hashing O(1) Constant time search using hash functions

Hash Function Concept

A hash function transforms a key into an address within a predefined address space:

Key → Hash Function → Address
h: U → {0,1,...,N-1}

Where U is the set of all possible keys, and N is the address space size

  • Home Address: The address generated by the hash function for a specific key
  • Synonyms: Different keys that map to the same home address
  • Collision: When two different keys generate the same home address

Hash Functions

Simple Hash Function Process

A typical hash function involves three steps:

  1. Transform key to number: Convert characters to ASCII codes
  2. Fold and add: Chop off pieces and sum them, then take modulus with a prime number
  3. Map to address space: Divide by address space size (preferably prime)
Example: Hashing "LOWELL"

1. ASCII: L(76) O(79) W(87) E(69) L(76) L(76) → 767987697676

2. Fold and mod: 767987697676 mod 19937 = 13,883

3. Final address: 13,883 mod 101 = 46

Advanced Hash Functions

Square and Take Middle

Key = 453 → 453² = 205209 → Extract middle = 52

Radix Transformation

Key = 453 (base 10) → Convert to base 11 = 382 → 382 mod 99 = 85

Best Practices:

  • Use prime numbers when dividing keys
  • Examine keys for natural patterns (e.g., sequential years)
  • Apply folding to preserve natural separation

Collision Resolution Techniques

Progressive Overflow (Linear Probing)

When a collision occurs, search sequentially for the next available slot:

Insertion Process
  1. Go to home address h(k)
  2. If free → place record
  3. If occupied → try next position (wrap around at end)

Hashing with Buckets

Store multiple records per hash address using buckets:

Bucket Structure
[2 | JONES | /////////// | ARNSWORTH]
Counter indicates 2 records in bucket of size 3
  • Bucket: Block of records corresponding to one hash address
  • Implementation: Each bucket contains counter for record tracking
  • Loading: Create file of empty buckets before storing records

Double Hashing

Use two hash functions when collisions occur:

Process
  1. First function h₁(k) determines home address
  2. If occupied, apply second function h₂(k) = c
  3. Add c to home address for overflow addresses
  4. Continue until empty slot found

Note: c should be relatively prime to table size N

Making Deletions

Search continues past tombstones but stops at empty slots

  • Insert tombstone when next record is occupied
  • New records can replace tombstones
  • Too many tombstones increases search length

Performance Considerations

Solutions for deteriorating performance:

  • Move records backward after tombstone deletion
  • Complete reorganization through re-hashing
  • Alternative collision resolution techniques
  • Monitor average search length for degradation
Performance Degradation Example

Key Takeaways

  • Hashing provides O(1) search time, making it the fastest searching technique for implementing indexes
  • Hash functions transform keys into addresses, with collisions occurring when different keys map to same address
  • Progressive overflow (linear probing) is simple but can lead to clustering and performance degradation
  • Bucket hashing allows multiple records per address, improving space utilization
  • Double hashing uses a second hash function to find overflow addresses, reducing clustering
  • Deletions require special handling (tombstones) to maintain search correctness
  • Performance monitoring is essential as too many tombstones or collisions degrade search efficiency
  • Prime numbers in hash functions help achieve better random distributions
