File Structures & Organization

Understanding File Structures, Data Processing, Computer Architecture, and File Management

Chapter Overview

This chapter provides a comprehensive introduction to file structures, covering definitions, operations, data processing concepts, computer architecture differences, and file management fundamentals. It establishes the foundation for understanding how data is organized, stored, and accessed in computer systems.

File Structure Definition

What is a File Structure?

A File Structure is a combination of representations for data in files and operations for accessing the data.

File Structure = Data Representation + Operations

Without file structures, applications wouldn't know how to store, retrieve, or manage data efficiently.

File Structure Capabilities

File structures allow applications to perform the following operations:

  • Read data – retrieve stored information
  • Write data – insert or add new information
  • Modify data – update or change existing records
  • Search data – find specific records based on criteria
  • Traverse data – read data in a particular order (e.g., sequentially)

File Structure Types

Sequential File Structure

Data stored one record after another (like a list). To find a record, you may need to scan from the beginning.

Example: Text file with students' names in order of entry

Indexed File Structure

An index (like a book's index) is created to quickly locate specific records.

Example: Library system searching books by ISBN

Hashed File Structure

Data placed in a location based on a hash function.

Example: Storing employee records using employee ID hash

Data Processing Concepts

Data Processing Components

From a computer science perspective, data processing involves three main aspects:

Storage of Data

How data is saved on storage devices (RAM, hard disk, SSD, cloud)

Organization of Data

How data is structured (files, tables, databases, indexes)

Access to Data

How data is retrieved or modified efficiently (searching, updating, querying)

Data Processing = Storage + Organization + Access

Data Structures vs File Structures

Aspect Data Structures File Structures
Storage Location Main Memory (RAM) Secondary Storage (Disk, SSD)
Speed Fast (electronic access) Slow (mechanical/electronic)
Persistence Temporary (lost when power off) Permanent (survives shutdown)
Size Small (limited by RAM size) Large (limited by disk space)
Cost Expensive per GB Cheap per GB
Examples Arrays, Linked Lists, Trees Sequential Files, Indexed Files, B-Trees

Key Insight

Both involve: Representation of Data + Operations for accessing data

Main difference: Data structures deal with data in main memory (fast, temporary), while file structures deal with data in secondary storage (slow, permanent).

Computer Architecture & Memory Hierarchy

CPU (Processor)

Registers: Very small storage inside CPU, extremely fast

Cache: Small but faster memory, stores frequently used data

Main Memory (RAM)

Located between CPU and secondary storage

Properties: Fast, Small, Expensive, Volatile

Holds data and programs while running

Secondary Storage (Disk, Tape, DVD-R, SSD)

Used for permanent storage of data and programs

Properties: Slow, Large, Cheap, Stable (non-volatile)

Speed Comparison Analogy

Main memory access: ~120 nanoseconds

Magnetic disk access: ~30 milliseconds

Analogy: Looking at book index (20 seconds) vs Going to library (58 days)

Scale: 1 second = 1000 milliseconds = 1,000,000 microseconds = 1,000,000,000 nanoseconds

File Management Fundamentals

Computer File Definition

A computer file is a named collection of data that exists on a storage medium (hard disk, CD, DVD, USB flash drive).

Files can contain: group of records, documents, photos, music, videos, emails, or programs.

File Naming Rules

Microsoft Windows
  • Case sensitive: No
  • Max length: 255 characters (including path)
  • Spaces allowed: Yes
  • Disallowed characters: * \ : < > | " / ?
Mac OS
  • Case sensitive: No
  • Max length: 255 characters
  • Spaces allowed: Yes
  • Disallowed character: : (colon)

File Extensions & Formats

File extension: Optional identifier separated by period (e.g., Paint.exe)

Extensions provide clues to file contents (e.g., .exe = executable program)

Common File Formats:

Text Documents
.txt, .docx, .pdf, .rtf
Spreadsheets
.xlsx, .xls, .ods
Graphics
.bmp, .jpg, .png, .gif
Audio/Video
.mp3, .wav, .mp4, .avi

Main Goal of This Course

Primary Objective

Minimize number of trips to the disk to get desired information.

Ideally get what we need in one disk access or as few disk accesses as possible.

Strategy for Efficiency

Group related information so we're likely to get everything needed with only one trip to disk.

Example: Name, address, phone number, account balance stored together

Locality of Reference

To achieve efficient file access, we need good file structure design based on:

  • Locality of Reference in Time: If data is accessed now, it's likely to be accessed again soon
  • Locality of Reference in Space: If one data item is accessed, nearby items are likely to be accessed soon
