Magnetic Storage Technology

Understanding Hard Disks, Storage Basics, Disk Architecture, and Magnetic Storage Systems

Chapter Overview

This chapter explores magnetic storage technology, focusing on hard disk drives, their components, data organization, capacity calculations, and access mechanisms. It also covers tape storage and the importance of understanding secondary storage for efficient file system design.

Storage Technologies Overview

Secondary Storage Importance

Secondary storage is different from main memory (RAM) and understanding it is crucial for designing good file systems:

  • RAM is fast but temporary (data lost when power off)
  • Secondary storage keeps data permanently
  • To design efficient file systems, we must understand how secondary storage works
Magnetic Storage

Stores data by magnetizing microscopic particles on disk/tape surfaces

  • Hard Disks (HDD)
  • Magnetic Tapes
  • Floppy Disks (obsolete)
Optical Storage

Uses laser light to read/write data on reflective surfaces

  • CD-ROM, DVD-ROM
  • Blu-ray
  • Read-only/write-once
Solid State Storage

Uses electronic circuits with no moving parts

  • Solid-State Drives (SSD)
  • USB Flash Drives
  • Memory Cards

Hard Disk Drives (HDD)

HDD Components

A hard disk drive contains several key components that work together to store and retrieve data:

Platter
Circular magnetic disk where data is stored
Spindle
Rotating shaft that holds and spins platters
Read/Write Head
Magnetic sensor that reads/writes data
Actuator Arm
Moves read/write head across platter
Track
Circular path on a platter
Sector
Smallest storage unit (512 bytes or 4KB)
Cylinder
Tracks aligned on top of each other
Cluster
Group of sectors (OS allocation unit)

How HDDs Work

Platters (Multiple Layers)
Tracks (Concentric Circles)
Sectors (Track Segments)
  • Platters spin at high speeds (5400-7200 RPM)
  • Read/write heads hover just microinches above platter surfaces
  • Heads magnetize particles to write data
  • Heads sense particle polarities to read data
  • All heads move together as a unit
  • Data accessed directly (random access)

Data Organization on Disk

Disk Structure Hierarchy

Data is organized in a hierarchical structure on hard disks:

Hierarchy from Smallest to Largest:
Bit → Byte → Sector → Track → Cylinder → Platter → Disk
  • Sector: Smallest addressable unit (usually 512 bytes)
  • Track: Concentric circles on platter surface
  • Cylinder: Set of tracks directly above/below each other
  • Cluster: Group of contiguous sectors (OS allocation unit)

Cylinder Concept

Cylinder: The set of tracks on a disk that are directly above/below each other

Key Advantage: All information on a cylinder can be accessed without moving the read/write arm (seeking)

This is why files are ideally stored in cylinders rather than spreading across different surfaces.

Disk Capacity Calculation

Calculation Formulas

Number of cylinders = Number of tracks in a surface
Track capacity = Sectors per track × Bytes per sector
Cylinder capacity = Number of surfaces × Track capacity
Drive capacity = Number of cylinders × Cylinder capacity

Example Calculation

Given Disk Characteristics:

  • Bytes per sector = 512
  • Sectors per track = 63
  • Tracks per cylinder = 16
  • Number of cylinders = 4092

Step 1: Track capacity

63 × 512 = 32,256 bytes

Step 2: Cylinder capacity

16 × 32,256 = 516,096 bytes

Step 3: Drive capacity

4092 × 516,096 ≈ 2.11 GB

File Size Calculation

Scenario: File with 50,000 records, each 256 bytes

1. Total file size:

50,000 × 256 = 12,800,000 bytes

2. Sectors needed:

ceil(12,800,000 ÷ 512) = 25,000 sectors

3. Tracks needed:

ceil(25,000 ÷ 63) = 397 tracks

4. Cylinders needed:

25,000 ÷ (63 × 16) ≈ 24.8 cylinders

Disk Access Cost

Seek Time
Time to move arm to correct track
~4-15 ms
Rotational Delay
Time for disk to rotate to correct sector
~4-8 ms
Transfer Time
Time to move data to/from disk surface
Depends on data size

Average Total Access Time = Average Seek Time + Average Rotational Delay + Transfer Time

Bottleneck: Moving the read/write arm (seeking) is the slowest part

Optimization: Store related data in same cylinder to minimize seeking

Magnetic Tape Storage

Tape Characteristics

  • Access Type: Sequential access only
  • Speed: Much slower than disks
  • Cost: Considerably cheaper than disks
  • Capacity: High capacity (modern LTO-9: 18TB per tape)
  • Usage: Backup storage and long-term archiving

Where Tape is Used Today

Backup Storage

Companies and banks use tape for data backup to prevent data loss

Long-term Archiving

Hospital records, university archives, government data storage

Disaster Recovery

Off-site storage for data restoration after disasters

Why Tape is Still Used

  • Low cost per terabyte (cheaper than HDD/SSD)
  • High storage capacity (up to 18TB per tape)
  • Long lifespan (up to 30 years reliable storage)
  • High security (offline, immune to cyberattacks)
  • Energy efficient (power only when in use)

Key Takeaways

  • Hard disks organize data in sectors, tracks, and cylinders for efficient access
  • The cylinder concept allows accessing related data without arm movement
  • Disk access time consists of seek time, rotational delay, and transfer time
  • Capacity calculations use: cylinders × surfaces × sectors/track × bytes/sector
  • Magnetic tape remains important for backups and archiving despite sequential access
  • Understanding disk organization is crucial for designing efficient file systems
  • Different storage technologies (magnetic, optical, solid-state) have different characteristics
  • File placement strategies should minimize disk access time, especially seeking
