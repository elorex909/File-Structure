Storage Technologies

Clusters, Fragmentation, Optical Storage, Solid State Storage, and Block Organization

Chapter Overview

This chapter explores advanced storage concepts including cluster management, fragmentation, optical storage technologies (CD, DVD, Blu-ray), solid state storage, and block organization on disks. It covers how operating systems manage file storage and different storage technologies.

Clusters, Extents, and Fragmentation

What are Clusters?

A cluster is a fixed number of contiguous sectors that the file manager allocates to files as the smallest allocation unit.

Key Points:

  • Cluster = Group of sectors
  • File manager allocates integer number of clusters to a file
  • Improves sequential access performance
  • Managed through File Allocation Table (FAT)

Internal Fragmentation

Cluster (1024 bytes)
Internal Fragmentation

Example: File size = 10 bytes
Cluster size = 1024 bytes (2 sectors)
Unused space = 1014 bytes
This wasted space is internal fragmentation

Internal Fragmentation occurs when the file size is not a multiple of the cluster size, leaving unused space in the last cluster.

File Allocation Table (FAT)

  • FAT contains location of each cluster in a file
  • Like a book index - helps OS find file data on disk
  • Tracks where each file starts
  • Tracks where next part of file is located
  • Enables efficient file access and management

Extents

An extent is a group of contiguous clusters.

Advantage: If file is stored in a single extent, seeking is done only once

Problem: If not enough contiguous clusters, file is divided into multiple extents

Organizing Tracks by Blocks

What is a Block?

A block is a group of records stored together and read from disk in one I/O operation.

Count Subblock
Number of bytes in block
Key Subblock (Optional)
Last key value for searching
Data Subblock
Actual file records

Blocking Factor (BF)

Blocking Factor = Number of records stored in each block

Formula: BF = Block Size ÷ Record Size

Example: Block size = 1000 bytes, Record size = 100 bytes

BF = 1000 ÷ 100 = 10 records per block

Advantages of Blocking:

  • Faster access: Fewer disk reads (one block at a time)
  • Less overhead: Reduces time in file access
  • Efficient for sequential reading: Works well with large files
  • No internal fragmentation: No wasted space within blocks
  • No record spanning: Records don't span two blocks

Solved Example

Given: Track size = 20,000 bytes, Overhead per block = 300 bytes, Record size = 100 bytes

Blocking Factor = 10

Data in block = 10 × 100 = 1000 bytes

Total block size = 1000 + 300 = 1300 bytes

Blocks per track = floor(20000 ÷ 1300) = 15

Records per track = 15 × 10 = 150

Blocking Factor = 60

Data in block = 60 × 100 = 6000 bytes

Total block size = 6000 + 300 = 6300 bytes

Blocks per track = floor(20000 ÷ 6300) = 3

Records per track = 3 × 60 = 180

Conclusion: Larger blocking factor (60) stores more records per track (180) than smaller blocking factor (10) which stores only 150 records.

Optical Storage Technology

How Optical Storage Works

Optical storage (CD, DVD, Blu-ray) stores data as microscopic pits (dark spots) and lands (lighter, non-pitted areas) on the disc surface.

Label Printed top surface showing title/design
Polycarbonate Substrate Main transparent base providing disc shape
Reflective Layer Reflects laser light for data reading
Recording Layer Stores data as pits and lands
CD (Compact Disc)
  • Original: 650 MB (74 minutes music)
  • Improved: 700 MB (80 minutes)
  • Red laser technology
  • Music, simple files
DVD (Digital Video Disc)
  • Single layer: 4.7 GB
  • Dual layer: 8.5 GB
  • Red laser technology
  • Movies, software
Blu-ray
  • Single layer: 25 GB
  • Dual layer: 50 GB
  • Blue-violet laser (shorter wavelength)
  • HD movies, large games

Optical Technology Categories

Read-only (ROM)

Factory-written data, cannot write to disc

Recordable (R)

Write once, then becomes read-only

Rewritable (RW)

Write, erase, and rewrite multiple times

Solid State Storage Technology

How SSD Works

Solid State Storage (flash memory) stores data in erasable, rewritable circuitry rather than on spinning disks or streaming tape.

  • Contains grid work of circuitry with cells
  • Each cell has two transistors acting as gates for 1s and 0s
  • Very little power required (ideal for battery devices)
  • No moving parts
Solid State Drive (SSD)
  • Substitute for hard disk drive
  • Same size as HDD or microprocessor chip
  • Fast access, no moving parts
  • Capacity: 32GB-256GB+
Memory Cards
  • Flat, solid state storage medium
  • Used in digital cameras, media players
  • Requires card reader
  • Various formats (SD, microSD, etc.)
USB Flash Drives
  • Portable storage with built-in connector
  • Capacity: 16MB to 256GB+
  • Slower than HDD but very portable
  • Plug-and-play convenience

SSD Advantages & Disadvantages

Advantages

  • Fast access (no moving parts)
  • Very durable (vibration resistant)
  • Low power consumption
  • Silent operation
  • Resistant to temperature changes

Disadvantages

  • Higher cost per GB
  • Limited write cycles
  • Smaller capacity than HDD
  • Data recovery more difficult

Key Takeaways

  • Clusters are the smallest allocation units managed by the file system through FAT
  • Internal fragmentation wastes space within clusters when files don't fill them completely
  • Block organization improves efficiency by reading multiple records in single I/O operation
  • Larger blocking factors generally store more data per track but require larger blocks
  • Optical storage uses pits and lands with different laser technologies (red for CD/DVD, blue for Blu-ray)
  • Solid state storage has no moving parts, making it faster and more durable than magnetic storage
  • Different storage technologies have trade-offs in cost, capacity, speed, and portability
  • Choosing cluster size and blocking strategy depends on file sizes and access patterns
