06

Managing Files of Records

Record Access, Keys, Sequential Search, Direct Access, and Header Records

Chapter Overview

This chapter focuses on techniques for accessing records within files, including different types of keys, search methods (sequential and direct), and the role of header records in file organization and management.

Record Keys

What is a Key?

A key is a subset of the fields in a record used to uniquely identify the record.

  • Keys correspond to fields or combinations of fields used in search operations
  • Not every field in a record is necessarily a key
  • Keys must have a canonical form (consistent representation)
  • Keys are not dataless - they contain actual data values
Primary Key
  • Must identify records uniquely
  • Not dataless (contains actual values)
  • Has canonical form
  • Example: Student ID, Employee Number
  • Used for exact record identification
Secondary Key
  • Does not identify records uniquely
  • Not dataless (contains actual values)
  • Has canonical form
  • Example: Student Name, Department
  • Used for searching/categorizing records

Key Characteristics

Uniqueness

Primary key must be unique, secondary key may have duplicates

Data Content

Both contain actual data values (not just pointers)

Canonical Form

Standardized representation for consistent comparison

Search Purpose

Primary: exact retrieval; Secondary: categorization/filtering

When to Use Sequential Search

  • ASCII files where you're searching for some pattern
  • Files with few records (e.g., 10 records)
  • Files that hardly ever need to be searched (e.g., tape files)
  • Files where you want all records with a certain secondary key value, where a large number of matches is expected
  • When the file is not sorted by the search key

Byte Offset Calculation

Formula:

Byte Offset = RRN × Record Length

Example:

RRN = 30 (31st record)
Record length = 101 bytes
Byte offset = 30 × 101 = 3030

Once the byte offset is calculated, the system uses the seek operation to go directly to that position in the file.

Performance Characteristics

O(1)
Constant time access
Direct access is possible when we know the Relative Record Number (RRN)

Requirements for Direct Access:

  • Records must be fixed length
  • Must know the RRN of the desired record
  • First record has RRN 0, next has RRN 1, etc.
  • Requires an index file or calculation mechanism

Header Records

What is a Header Record?

A header record is placed at the beginning of a file and is used to store information about the file contents and the file organization.

Record Length Length of data records in bytes
Last Update Date and time of most recent file update
File Name Name of the file for identification
Organization Type Sequential, indexed, direct, etc.
Record Count Number of records in the file

Header Record Characteristics

  • Placed at the beginning of the file (position 0)
  • Usually has a different structure and size than data records
  • Contains metadata about the file
  • Read first when opening a file
  • Updated when file contents change
  • Essential for file management and organization

Key Takeaways

  • Primary keys uniquely identify records while secondary keys enable searching and categorization
  • Sequential search has O(n) time complexity and is suitable for small files or infrequent searches
  • Direct access provides O(1) constant time access but requires fixed-length records and known RRN
  • Record blocking improves sequential search performance by reducing disk I/O operations
  • Header records store metadata about files and are essential for proper file management
  • Byte offset for direct access is calculated as: RRN × Record Length
  • Choosing between sequential and direct access depends on file size, access frequency, and record organization
  • Both primary and secondary keys have canonical forms and contain actual data values
