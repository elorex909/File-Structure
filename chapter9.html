Lossy Compression Methods

JPEG Image Compression, MPEG Video Compression, and Audio Encoding Techniques

Chapter Overview

This chapter explores lossy compression methods that permanently remove some data to achieve significant size reduction. Unlike lossless methods, lossy compression sacrifices perfect accuracy for practical benefits in images, video, and audio files where humans cannot easily detect the missing information.

Lossy vs. Lossless Compression

What is Lossy Compression?

  • Permanently removes data to reduce file size
  • Removes data humans cannot easily detect (subtle visual/audio details)
  • Original ≠ Decompressed (some information is lost forever)
  • Used for: Images, video, audio files
  • Examples: JPEG, MPEG, MP3

Why Accept Data Loss?

Human Vision Limits

Eyes can't detect subtle color changes or fine details at certain resolutions

Human Hearing Limits

Ears can't hear very high frequencies or quiet sounds masked by louder ones

  • Cheaper storage - smaller files need less space
  • Faster transmission - smaller files transfer quicker
  • Less processing time - smaller files are faster to handle
  • Acceptable quality - loss is imperceptible to humans
  • Practical for media - essential for web images, streaming video, digital music

Lossy Compression Methods

JPEG

Joint Photographic Experts Group

  • Compresses pictures and graphics
  • Divides image into 8×8 pixel blocks
  • Uses DCT transform + quantization
  • Removes high-frequency details
  • File extensions: .jpg, .jpeg
MPEG

Moving Picture Experts Group

  • Compresses video files
  • Video = sequence of images (frames)
  • Spatial + temporal compression
  • Stores differences between frames
  • File extensions: .mpg, .mpeg, .mp4
MP3/Audio

Audio Compression

  • MP3: MPEG Audio Layer 3
  • Speech: 64 kHz → compressed
  • Music: 1.411 MHz → compressed
  • Uses psychoacoustic model
  • File extensions: .mp3, .aac

JPEG Encoding Process

1

Divide into 8×8 Blocks

Grayscale image divided into 8×8 pixel blocks to reduce calculations

2

Apply DCT (Discrete Cosine Transform)

Converts spatial pixel values into frequency domain

3

Quantization (LOSSY STEP)

Divide DCT values by quantization table, drop decimals

4

Zigzag Scan

Read 8×8 block diagonally to group zeros together

5

Lossless Compression

Apply RLE then Huffman coding to compress remaining data

DCT (Discrete Cosine Transform)

DCT converts 64 pixel values (spatial domain) into frequency values:

2-D DCT Formula (JPEG uses N=8):

Xu,v = α(u)α(v) ∑x=0N-1 ∑y=0N-1 f(x,y) cos[π(2x+1)u/(2N)] cos[π(2y+1)v/(2N)]

Where α(k) = 1/√N for k=0, α(k) = 2/√N for k>0

Low Frequencies

Represent smooth areas, gradual color changes

High Frequencies

Represent fine details, edges, noise

Goal: Keep important low-frequency values, reduce unimportant high-frequency values

MPEG Video Compression

Video = Sequence of Frames

Video is simply a rapid sequence of images (frames).

Spatial Compression

Inside each frame
Each frame compressed like JPEG image

Temporal Compression

Between frames
MAJOR SAVINGS - only store differences

Temporal Compression Concept

Key Insight: Most frames are very similar to each other.

Frame 1

Person sitting
Mouth closed
Background static

Store complete frame

Frame 2 (0.04s later)

Person sitting
Mouth slightly open
Background static

Store ONLY mouth region difference

Instead of storing 100% of Frame 2, MPEG stores only the changed pixels (≈5-10% of frame).

MPEG Compression Savings

  • Spatial compression alone: 10:1 ratio (JPEG on each frame)
  • Temporal compression adds: 30:1 to 50:1 additional ratio
  • Total compression: 300:1 to 500:1 possible
  • Example: Uncompressed HD video: ~1 Gbps → MPEG-4: 2-8 Mbps
  • Enables video streaming, digital TV, DVD/Blu-ray storage

Audio Compression

Predictive Encoding

Used for: Speech

Stores only differences between samples

Samples: 100, 101, 102, 103, 103, 104
Differences: +1, +1, +1, 0, +1

Standards: GSM (13 kbps), G.729 (8 kbps), G.723.3 (6.4 kbps)
Perceptual Encoding (MP3)

Used for: Music

Removes sounds humans cannot hear:

  • Very high frequencies
  • Quiet sounds masked by louder ones
  • Uses psychoacoustic model
CD Audio: 1,411 kbps → MP3: 128 kbps (11:1 compression)

Key Takeaways

  • Lossy compression permanently removes data humans can't easily detect
  • JPEG uses 8×8 blocks, DCT transform, quantization (lossy step), then lossless compression
  • DCT converts spatial pixels to frequency: low frequencies = smooth areas, high frequencies = details
  • Quantization divides values and drops decimals → creates many zeros for compression
  • MPEG compresses video with spatial (JPEG per frame) + temporal (store differences) compression
  • Audio compression: Predictive for speech (store differences), Perceptual (MP3) for music (remove inaudible sounds)
  • Typical ratios: JPEG 10-20:1, MPEG 50-200:1, MP3 10-12:1, Speech codecs 100-200:1
  • Lossy enables practical media storage/streaming: web images, YouTube, Spotify, digital TV
